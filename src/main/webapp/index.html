Why not RTC?

There is no need to move away from RTC if you are happy with it. We decided to move to git and JIRA for the following reasons:

  • performance
  • stability
  • offline capabilities
  • integration into various IDEs
  • integration into build tools
  • community support

Git

Learn how to migrate to git using rtc2git script or cli command by checking out our github pages for the cli extension or the script

JIRA

Learn how to migrate your existing work items to another issue tracker using rtc2jira by checking out our github page.
rtc2jira is still in development.
Feel free to contribute by opening a pull-request or an issue.
